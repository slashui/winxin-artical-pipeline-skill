# 正文插图 Prompt 模板

## 用法

生成正文插图时，按下面结构拼接：

1. 内容段
2. 固定风格段
3. 负面约束

最终格式：

```text
[内容段]

[固定风格段]

[负面约束]
```

---

## 内容段模板

```text
这是一张用于解释[主题]的正文配图。图片需要让读者一眼看懂[核心关系/核心结论]。画面中应包含[关键元素]，结构上突出[层级/流程/对比/系统关系]，适合作为文章中段的解释型插图。
```

### 内容段示例 1：总览图

```text
这是一张用于解释“微信公众号文章生产与发布流程”的正文配图。图片需要让读者一眼看懂从原始素材到公众号草稿箱的完整工作流。画面中应包含原始素材输入、文案生成、图片规划、图片生成、回填文章、上传草稿箱这六个步骤，结构上突出顺序流程和产物流转，适合作为文章中段的解释型插图。
```

### 内容段示例 2：概念图

```text
这是一张用于解释“AI 编程中的致命三重奏”的正文配图。图片需要让读者一眼看懂私密数据、恶意指令、数据外传三者同时存在时的风险。画面中应包含三角结构、AI 助手、邮箱和数据泄露符号，结构上突出危险关系，适合作为文章中段的解释型插图。
```

### 内容段示例 3：流程图

```text
这是一张用于解释“红绿 TDD 如何提升 AI 代码质量”的正文配图。图片需要让读者一眼看懂先写测试、先失败、再实现、再通过的闭环。画面中应包含测试、失败状态、实现过程、通过状态和循环箭头，结构上突出反馈回路，适合作为文章中段的解释型插图。
```

## 固定风格段

```text
横版 16:9，适合微信公众号正文配图。clean infographic poster，presentation slide style，editorial illustration，minimal vector composition，clear visual hierarchy。整体强调“解释清楚”而不是“装饰好看”，适合手机阅读。优先使用简洁图形、结构化布局、强信息层级和清晰对比，让读者一眼看懂文章接下来要展开的内容。所有可见标题、标签、说明文字都必须使用简体中文。风格现代、克制、专业，不要照片感，不要 3D 渲染，不要真实 UI 截图。推荐使用深蓝、橙色、米白色作为主色，局部可加入青色或灰色辅助，保持知识型媒体插图气质。
```

---

## 负面约束

```text
不要照片感，不要 3D 渲染，不要影视海报风，不要赛博朋克，不要复杂背景噪音，不要过度装饰，不要乱码文字，不要英文标签，不要真实软件界面截图，不要花哨渐变，不要难以阅读的小字，不要低清晰度，不要无关人物。
```
